## 📝 Notas detalhadas (uso interno / GitHub release)

### 🐛 Correções

- **Filtro de cidade com acento (Anúncios, Utilidades, Vagas, Eventos, Achados, Destaques, Premium)**: o painel web gravava `cidade_normalizada` sem acentos e o app comparava com a cidade detectada do GPS (com acentos). O resultado era anúncios de "Rondonópolis" não aparecerem para usuários de Rondonópolis, enquanto cidades sem acento (Toledo, Goiânia escrito sem til) funcionavam. Agora a comparação faz o match ignorando acentos dos dois lados (`LocationService._chaveCidadeComparacao`).

### 🔄 Atualização obrigatória do app

- **Leitura forçada do servidor**: antes, em redes lentas ou após primeira abertura offline, o serviço podia ler `configuracoes/atualizacao_app` do cache local do Firestore e continuar com a versão mínima antiga. Agora a leitura usa `Source.server` com fallback para cache apenas quando o servidor não responde.
- **Logs mais claros** no console durante o splash: informam exatamente qual versão está instalada vs. qual versão mínima foi lida do Firestore e o resultado da comparação (`[atualizacao_app] instalada=X mínima=Y comparação=Z`). Facilita diagnóstico quando o lojista master alterar a versão mínima.
- **Tela Sobre reflete automaticamente a nova versão** via `PackageInfo.fromPlatform()` — após este build, aparece `1.2.0`.

### 🔧 Como usar a atualização obrigatória

No Firebase Console → Firestore → `configuracoes/atualizacao_app`:

- `versao_minima_android`: string semver, ex. `1.2.0`. Qualquer aparelho rodando versão menor é bloqueado com tela "Atualização necessária".
- `versao_minima_ios`: idem para iOS (opcional; exige `url_loja_ios` preenchido).
- `url_loja_android` / `url_loja_ios`: links das lojas (opcional — se vazio, o Android monta automaticamente com o package do app).
- `mensagem`: texto extra opcional exibido no diálogo.

Exemplo para forçar atualização da 1.1.0 → 1.2.0:

```
versao_minima_android: "1.2.0"
```

Usuários em `1.2.0+` continuam abrindo normalmente. Usuários em `1.1.0` veem a tela de bloqueio com botão "Atualizar na loja".
